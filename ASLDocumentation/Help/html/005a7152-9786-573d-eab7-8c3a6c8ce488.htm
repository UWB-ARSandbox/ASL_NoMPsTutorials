It should be noted that the order in which a packet is created matters. The order should be as follows so that it can be decoded properly. Amount of data pieces (e.g., 2), the lengths in bytes of these data pieces (e.g., 4,4), and the data themselves converted to byte format (e.g., 4,4). The Combing functions found here automatically add this meta data for you. If you follow how ASLObject functions already create these data packets then you will be following the correct formatting. The important thing to remember is that however you encode a packet, you must remember to decode in the same manner.
